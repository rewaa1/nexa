# Transition Philosophy

## Core Principle

Transitions are not decorations.

Transitions are part of the storytelling.

The user should never feel like they are scrolling between sections.

The user should feel like they are travelling through a living system.

---

## Rule 1

Never reuse the same transition style repeatedly.

Avoid:

Hero → fade

Section → fade

Section → fade

Section → fade

Repeating transition patterns creates predictability.

Predictability creates boredom.

---

## Rule 2

Every major section should have its own transition identity.

Examples:

Hero → Orbital Zoom

Services → Gravitational Pull

Projects → Planet Formation

Process → Data Stream Navigation

Contact → Docking Sequence

Each transition should feel unique and tied to the narrative.

---

## Rule 3

Transitions should evolve.

The website should become more ambitious as the journey progresses.

Early transitions:

- subtle
- elegant
- mysterious

Later transitions:

- immersive
- cinematic
- technically impressive

---

## Rule 4

Transitions should occasionally surprise the user.

Examples:

- camera rotation
- dimensional shifts
- object morphing
- orbit collapse
- particle reconstruction
- scene transformation
- environmental changes

Unexpected moments create memorability.

---

## Rule 5

Transitions should communicate meaning.

Example:

Do not rotate a planet simply because it looks cool.

Rotate it because the user is moving into a new orbit.

Every movement must have narrative purpose.

---

## Forbidden

- Repeated fade transitions
- Repeated slide-up animations
- Repeated scale-in animations
- Repeated card reveals
- Repeated section wipes
- Copy-paste transition systems

---

## Orbix Standard

When creating a new section:

Do not ask:

"How should this section animate?"

Ask:

"How should the universe transform when entering this section?"

The transition should be designed before the section itself.
